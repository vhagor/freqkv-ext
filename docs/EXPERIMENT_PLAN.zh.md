### 关键证据（E2 attn relerr，越低越好；小波几乎全胜）

| γ | DCT | DFT-RoPE | Wavelet | RST(best α) | 备注 |
|--:|----:|---------:|--------:|------------:|------|
| 0.5 | 0.27–0.35 | ≈DCT | **0.10–0.17** | 0.23–0.40 | 小波大幅领先 |
| 0.25 | 0.39–0.54 | ≈DCT | **0.28–0.33** | 0.36–0.59 | 小波领先 |
| 0.125 | 0.46–0.61 | ≈DCT | **0.42–0.46** | — | 小波领先 |
| 0.03125 | ~0.55–0.66 | ≈DCT | ~0.49–0.62 | — | 极端压缩处差距收窄 |

- DFT-RoPE 与 DCT 处处相差 <0.03 → 调制定理预言的"酉等价"被精确印证，**DFT 在精度上无 headroom**。
- V 比 K 更难压（DCT V relerr 0.71 @γ=0.5），小波对 V 收益同样最大。
- 绝对误差偏高是因为离线一次性压全 2048、无 sink+recent 窗口；码间排序仍有效。

### 方向转向

1. **DFT-RoPE / RST 退出精度赛道**（数学上注定 ≲ DCT）。仅可能作为**独立的解码加速**
   贡献（C3，post-RoPE caching + Parseval），且需自证 FLOP 收益，否则删。
2. **小波成为方法主体**。但先用 NE2' 拆清"基底效应 vs 自适应选择效应"——若自适应
   DCT（top-k 而非低通）能追平大半，则方法应是"自适应选择 + 可选小波基"，更省且保留
   FreqKV 的快变换。
3. **C1 频率梳 + 平滑性刻画** 作为测量/理论贡献（Figure 1 + 为何小波更优的机理）。
4. **E3 异质性 salvage**：把"per-pair 注水"推广为 **per-channel/per-layer 自适应预算**，
   嫁接到小波（基底无关的率失真分配）。

### 候选新方法：WaveKV

> 序列轴**自适应小波**压缩 + **per-channel/per-layer 率失真预算分配**；正交小波满足
> Parseval ⇒ 可做**压缩域注意力**（给小波也补上系统侧故事，替代 DFT 的 C3）；真实部署
> 用**稀疏系数 cache** + FreqKV 的 **sink + recent window 迭代**设置。

机理叙事（待 NE2' / NE-Smooth 坐实）：**K 沿序列轴是分段光滑 / 有界变差信号（局部光滑 +
语义边界处突变），不是平稳 AR(1)**；故边缘自适应的小波基胜过平稳信号最优的 DCT。

## 0. 原创性定位：我们到底新在哪

### 0.1 一句话框架

> **把 post-RoPE 的 KV 沿序列轴分解成两部分：一个由 RoPE 决定的、位置可预测的
> "频率梳"主体（spectral bulk），加上一个时域/小波域的稀疏残差（sparse
> residual）。前者用每对 pair 的 RoPE 匹配带通廉价编码，后者用稀疏系数捕获被
> 平滑主体漏掉的局部事件。两者之间按率失真做预算分配。**

我们把这个方法暂命名为 **RST-KV（RoPE-Spectral Transform coding with sparse
residual）**。

### 0.2 为什么这不是"已有成果的杂糅"

| 维度 | 已有工作 | RST-KV 的新点 |
|------|----------|----------------|
| 频域 KV 压缩 | FreqKV：**均匀** DCT 低通，pre-RoPE | 用调制定理证明"相关频带是**每对 pair 平移**的"，并据此做 RoPE 匹配带通 |
| 局部事件 | 所有 transform-coding 方法都把局部事件当高频丢掉 | 显式建模为**稀疏残差**，与主体分离编码 |
| Token 级压缩 | SnapKV / H2O：丢 token | 我们不丢 token，压的是每个 token 的序列轴表示 |
| 通道级压缩 | PALU：hidden 维低秩 | 正交方向：我们压序列维 |
| 统一性 | —— | 把"DFT vs 小波"从二选一改写成"一个码的两部分"，并用率失真统一 |

**核心论点**：DFT-RoPE 和小波不是竞争者，而是同一个最优码的两个互补分量。这个
"spectral bulk + sparse residual"分解在 KV 压缩里没有先例，是本工作的原创内核
（与 robust PCA 的 low-rank + sparse 分解同构，但发生在 RoPE 频域）。

### 0.3 三条可证伪的主张

- **C1（测量/理论）**：post-RoPE 的 K 沿序列轴是一个**确定性频率梳**，第 $i$ 对
  pair 的能量峰在 $n_i = \mathrm{round}(\theta_i N / 2\pi)$，与输入无关、与模型
  弱相关、随 RoPE base 可预测地缩放。
  *现状*：LLaMA-2-7B 上已确认（pair 0 → bin 652，梳齿结构清晰）。

- **C2（方法）**：RST-KV 的"带通主体 + 稀疏残差 + 率失真分配"在**相同压缩预算**
  下，重建/注意力输出误差严格低于 FreqKV 的均匀 DCT 低通，且差距在高压缩区
  （$\gamma \lesssim 0.16$）和局部检索任务（needle / 代码）上显著放大。

- **C3（系统）**：post-RoPE caching + Parseval 压缩域注意力 + IDFT 融进 $W_o$，
  能在长上下文解码时消去在线 RoPE GEMM 和 IDFT，降低 decode 延迟。

## 2. E1 — 频谱普适性刻画（扩展已有结果）

**目的**：把 C1 从"LLaMA-2-7B 上成立"升级为"RoPE 模型的结构性现象"。

**做什么**（在 `analyze_spectrum.py` 基础上扩展）：

1. **跨模型**：LLaMA-2-7B（base=1e4，已有）、LLaMA-3-8B（base=5e5）、
   Mistral-7B（base=1e6）。预测：base 越大，$n_i = \theta_i N / 2\pi$ 越小，梳齿
   越向 DC 收拢。**如果三个模型的梳齿都按各自 base 预测的位置出现，C1 就从"单点
   观察"变成"随 base 缩放的定律"**——这是强得多的证据。
2. **峰锐度（Q 因子）**：对每对 pair 拟合峰宽，量化"要保住该 pair 需要多少带宽
   $L_i$"。这直接喂给 E3 的分配算法。
3. **能量包络**：量化"梳齿能量 / DC 能量"比值随层、随 pair 的变化（图已显示 DC
   主导 2–3 个数量级，要给出确切数字）。

**产出**：跨模型频谱对比图（论文 Figure 1）+ 每对 pair 的 $(n_i, Q_i, \text{能量
占比})$ 表。

**证伪**：若换 base 后梳齿位置不变 / 不按 $\theta_i$ 移动 → C1 垮，整条线回退到
"FreqKV 复现"。

## 3. E2 — 离线率失真（最高性价比的原创实验）

**目的**：在**完全不训练**的前提下，直接在真实抓取的 K/V 上画出率失真曲线，验证
C2 的核心——混合码（bulk + residual）严格优于 DCT。

**为什么这个实验最关键**：
- 训练无关，单卡 2 小时能跑完，**在花 H100 训练之前就能判生死**。
- 直接产出论文的核心率失真图（Figure 2），是 C2 最干净的证据。

**做什么**（需新写 `scripts/rate_distortion.py`）：

对每个压缩比 $\gamma \in \{0.5, 0.25, 0.125, 0.0625, 0.03\}$，在抓取的真实
K/V 上比较 4 种编码的两类误差：

1. **K 重建误差**：$\|K - \hat{K}\|_F / \|K\|_F$。
2. **注意力输出误差**（更重要）：
   $$
   \frac{\big\|\, \mathrm{softmax}(qK^\top)V - \mathrm{softmax}(q\hat{K}^\top)\hat{V} \,\big\|}{\big\|\mathrm{softmax}(qK^\top)V\big\|}
   $$
   因为最终影响模型的是注意力输出，不是 K 本身。这是比 FreqKV 论文更严格的度量。

4 种编码：
- **DCT 均匀低通**（FreqKV 基线）。
- **DFT-RoPE 带通**（只有 bulk）。
- **小波硬阈值**（只有 residual 思路）。
- **RST 混合**：带通 bulk（占预算 $\alpha\gamma$）+ 时域稀疏残差（占 $(1-\alpha)\gamma$），
  其中残差 = $K - \text{bulk 重建}$ 取最大幅度的若干项。

**证伪**：若 RST 混合在注意力输出误差上**不**优于 DCT（尤其在小 $\gamma$），则
"bulk + sparse residual 分解"不成立，C2 主体被推翻——此时退回到纯小波或纯
DFT-RoPE 的单分量方法，原创性下降但仍可发表为"RoPE 频域刻画 + 小波基"。

**额外原创产出**：画"最优 $\alpha$（bulk/residual 预算分配比）随 $\gamma$ 的变化"
曲线。预期：$\gamma$ 大时 $\alpha \to 1$（带通够用），$\gamma$ 小时 $\alpha$ 下降
（残差变重要）。这条曲线本身就是一个新发现。

## 4. E3 — 每对 pair 的率失真预算分配

**目的**：把"均匀 $\gamma$"升级为"按测得能量做最优分配"，证明 FreqKV 的均匀分配
是次优特例。

**做什么**：
- 用 E1 测得的每对 pair 能量谱，按经典 Shannon 注水（water-filling）解每对 pair
  的最优带宽 $L_i$，约束 $\sum_i L_i = \gamma N \cdot (d/2)$。
- 对比：均匀分配（每对都 $L$）vs 注水分配，在 E2 的注意力输出误差度量上比较。

**产出**：分配热力图（哪些 pair / 哪些层拿到更多带宽）——这是一张有信息量的原创
图，揭示模型把"位置预算"花在哪里。

**证伪**：若注水分配 ≈ 均匀分配 → 说明每对 pair 能量其实差不多，分配这个子问题
没价值，砍掉 E3，C2 退化为"固定带通"。

## 5. E4 — 训练 + 下游评测（H100 主力）

**前置 gate**：仅当 E2 显示 RST 混合在离线率失真上占优才进入。

**训练变体**（砍掉 `dft_lowpass`）：
- `dct`：FreqKV 基线（锚点，必跑）。
- `dft_rope`：纯带通（C2 的 bulk 分量单独验证）。
- `wavelet`：纯小波（residual 思路单独验证）。
- `rst`：混合码（主力，E2 选出的最优 $\alpha$）。

**评测**：
- **PPL γ 扫**：$\gamma \in \{0.5, 0.25, 0.125, 0.0625\}$，PG-19 + Proof-pile，
  长度 8K/16K/32K。**重点不是 $\gamma=0.5$ 单点**（图已说明那里没区别），而是
  $\gamma \le 0.125$ 的高压缩区，预期 RST > dft_rope > dct，wavelet 在 Proof-pile
  上有戏。
- **LongBench**：按子任务分析（QA / 代码 / 摘要）。
- **Needle-in-a-Haystack**：1K–16K × 5 深度。**这是 residual 分量的决定性测试**：
  needle 是教科书级的局部事件，RST 的稀疏残差应当在 >8K 显著拉回 DCT 的掉点。

**证伪**：若 γ 扫全程 RST 不优于 DCT，且 Needle 上无改善 → C2 在端到端被否定，
项目以 E1+E2 的"刻画 + 离线 RD"作为结论收尾（仍是一篇 measurement/analysis
论文，但不是 method 论文）。

## 6. E5 — 压缩域注意力（系统层原创点，C3）

**目的**：证明这条线不只是精度故事，还有 DFT 专属的系统收益——这是小波给不了的。

**做什么**：
- 实现 post-RoPE K 直接 cache（DFT 域 RoPE 可逆，已在 `dft_rope_aware` + patch
  里铺好）。
- Parseval 压缩域注意力：$\langle q,k\rangle = \frac1N\sum_\omega Q[\omega]K^*[\omega]$，
  在保留的 $L$ 个 bin 上算，跳过 IDFT。
- V 侧把 IDFT 离线融进 $W_o$（PALU 式），推理期完全无 IDFT。

**度量**：长上下文 decode 的 TTFT、per-token 延迟、吞吐，对比 FreqKV 和 dense。

**证伪**：若在 16K–32K 上下文上无可测加速 → C3 不成立，系统贡献删除，论文专注
C1+C2。

## 7. E6 — 消融与负面结果（保证科学诚实）

- **$\alpha$ 消融**：bulk/residual 预算比扫描，验证 E2 的最优 $\alpha$ 曲线在训练
  后依然成立。
- **残差基消融**：时域 top-k vs 小波域 top-k vs DCT 高频残差——确认"稀疏残差"该
  用哪个域。
- **逐层分配消融**：所有层同 $\gamma$ vs 按层分配。
- **sink / recent window 消融**：对齐 FreqKV 的迭代压缩设置。
- **负面结果登记表**：E1–E5 每处期望落空都记录原因，写进 Limitations。

## 11. 立即可做（不占 H100）

按性价比排序，下面三件都在单卡/CPU 完成，且决定 H100 是否值得投：

1. **E1 扩展**：把 `analyze_spectrum.py` 加上跨模型 + Q 因子 + 能量占比统计。
2. **E2 新脚本** `scripts/rate_distortion.py`：抓 K/V → 4 码重建 → 注意力输出误差
   → 率失真曲线 + 最优 $\alpha$ 曲线。**这是下一步最该写的代码**。
3. **E3 原型**：在 E1 的能量谱上跑注水分配，出分配热力图。

E2 一旦显示 RST 混合占优，C2 的核心就在训练之前立住了，H100 的训练才有意义。
